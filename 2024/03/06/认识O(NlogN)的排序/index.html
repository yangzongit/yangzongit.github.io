认识O(NlogN)的排序

1.剖析递归行为

递归过程本质上可以理解为是一颗多叉树，计算树的所有节点就是利用栈进行遍历，每个节点都通过自己的子节点给自己汇总信息后再往上返回。栈空间就是树的高度，故只用在一个高度压栈。

​​

2.递归行为的时间复杂度估计

master公式：T(N) = a*T(N/b) + O(N^d)​

T(N):母问题的数据量是N级别的​

T(N/b)：子问题的样本量，是等量的​

a：子问题被调的次数​

O(N^d)：除去调用之外剩余的过程的时间复杂度​

满足如上公式的程序都可以根据master公式计算时间复杂度：

  • ​log(b，a) > d ​：时间复杂度为O(N^log(b，a))​
  • ​log(b，a) = d ​：时间复杂度为O(N^d * logN)​
  • ​log(b，a) < d ​：时间复杂度为O(N^d)​

3.归并排序（时间复杂度为O(NlogN),空间复杂度为O(N)),符合master公式

归并排序这种O(NlogN)的算法比O(N²)的算法好的实质是：O(N²)的算法浪费了大量的比较行为，例如插入、冒泡、选择排序算法。而归并排序，左侧和右侧的比较行为并没有浪费，而是变成规整成一个有序的整体以后再去跟更大的整体进行比较，所以这种比较行为是向上传递信息的。所以归并排序能做到O(NlogN)的时间复杂度。

4.归并排序的拓展:

1）小和问题：

描述
在一个数组中，每一个数左边比当前数小的数累加起来，叫做这个数组的小和。求一个数组的小和。

例子
[1,3,4,2,5]
1左边比1小的数：没有
3左边比3小的数：1
4左边比4小的数：1,3
2左边比2小的数：1
5左边比5小的数：1,3,4,2
所以小和为1+1+3+1+1+3+4+2=16

但是和经典归并排序merge的过程不一样，当左组的数和右组的数相等的时候，一定要拷贝右组的数

2)逆序对问题:

描述：

对于一个包含N个非负整数的数组A[1..n]，如果有i < j，且A[ i ]>A[ j ]，则称(A[ i] ,A[ j] )为数组A中的一个逆序对。

就是一个数组中，左边的数比右边的数大，那这俩个数就构成一个逆序对

例子：

数组（3，1，4，5，2）的逆序对有(3,1),(3,2),(4,2),(5,2)，共4个。

逆序对问题与小和问题思想一样，小和问题是求右边有多少个数比左边大，逆序对问题是左边有多少个数比右边大

在有序的情况下，左侧的当前下标的值比右侧当前下标的值大，那就说明左侧的当前下标之后的值（包括当前下标）都比右侧当前下标的值大

逆序对的数量就是(M - L + 1);

‍

5.快速排序

1）荷兰国旗问题

问题一
给定一个数组arr，和一个数num，请把小于等于num的数放在数组的左边，大于num的数放在数组的右边。要求额外空间复杂度O(1)，时间复杂度O(N)

实质是，在i下标前面分俩个区域，一个是小于等于区域，一个是大于区域。i下标的值小于等于num时，小于等于区域+1；i下标的值大于num，i++，大于区域+1。就是小于等于区域推着大于区域走，遇到不同情况区域+1，直到小于等于区域和大于区域占领整个数组就结束。

‍

问题二(荷兰国旗问题)
给定一个数组arr，和一个数num，请把小于num的数放在数组的左边，等于num的数放在数组的中间，大于num的数放在数组的右边。要求额外空间复杂度O(1)，时间复杂度O(N)

实质是，在i下标前是小于区域和等于区域，i下标后是待处理区域和大于区域。i下标的值小于num时，小于区域+1，小于区域+1的同时推着等于区域往右走；i下标的值等于num时，等于区域+1；i下标的值大于num时，大于区域+1，大于区域+1的同时和小于等于区域挤压着待处理区域，直到待处理区域为0，算法结束。

‍

2）快速排序1.0（时间复杂度为O(N²)）

3）快速排序2.0（时间复杂度为O(N²)，按最差的情况计算，每次partition只能排序一个数字）

4）快速排序3.0

快排最好的情况就是，num经过partition打在中间的位置，这个时候左右俩边递归的区域是相等的，根据master公式，时间复杂度为O(NlogN)；差的情况就是，某一边递归的区域特别大或者某一边递归的区域特别小。例如：1，2，3，4，5，6，7，8，9。

那么如果从partition的区域中等概率取出一个数放到区域最后面作为num，那么好情况和坏情况就变成了随机事件，而每一种情况都有一个时间复杂度。因为选取每一个位置都是等概率事件，所以每一个master公式出现也是等概率事件（权重1/N），把所有mastr公式求概率累加再求数学上的长期期望得出时间复杂度为O(N*logN) ，额外空间复杂度为O(logN) 。空间复杂度差情况就是O(N)有N层递归空间，好的情况就是每次划分值在中间，为O(logN)，一样根据数学期望得出额外空间复杂度为O(logN ) 。此时递归类似完全二叉树展开
