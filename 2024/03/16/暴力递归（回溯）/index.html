暴力递归（回溯）

1.暴力递归就是尝试
①把问题转化为规模缩小了的同类问题的子问题（一个问题拆成小问题去解决，小问题再去拆成更小的问题）
②有明确的不需要继续进行递归的条件(base case)
③有当得到了子问题的结果之后的决策过程(就是得到了子问题的结果怎么处理）
④不记录每一个子问题的解

2.汉诺塔问题

打印n层汉诺塔从最左边移动到最右边的全部过程

假设杆子为from，to，other。i层汉诺塔要从from去to，other是其他杆，大问题可以分解为小问题：

①把1~i-1层放到other上

②把i从from放到to上

③把1~i-1层从other上放到to

对于把1~ i-1层放到other上，这个时候i-1层的from不变，to变成了other，而1 ~ i-1的other是to了

暴力递归尝试的时候，给所有的过程定一个统一的标准，只要大标准做到了，底下的就是对的。 像上面的汉诺塔问题，放i层的时候先把i-1层全部放到other上，再把第i层从from放到to，最后把i-1层从other放到to。这个大标准做到了并且下面的每一层都做到了统一的标准，就是对的。

给定一个整型数组arr，代表数值不同的纸牌排成一条线。玩家A和玩家B依次拿走每张纸牌，规定玩家A先拿，玩家B后拿，但是每个玩家每次只能拿走最左或最右的纸牌，玩家A和玩家B都绝顶聪明。请返回最后获胜者的分数。
【举例】arr=[1.2.100.4]。
开始时，玩家A只能拿走1或4。如果开始时玩家A拿走1，则排列变为[2.100.4]，接下来玩家B可以拿走2或4，然后继续轮到玩家A…
如果开始时玩家A拿走4，则排列变为[1.2,100]，接下来玩家B可以拿走1或100，然后继续轮到玩家A..
玩家A作为绝顶聪明的人不会先拿4，因为拿4之后，玩家B将拿走100。所以玩家A会先拿1让排列变为[2.100.4]，接下来玩家B不管怎么选，100都会被玩家 A拿走。玩家A会获胜分数为101。所以返回101。

arr=[1.100.2]。
开始时，玩家A不管拿1还是2，玩家B作为绝顶聪明的人，都会把100拿走。玩家B会获胜分数为100。所以返回100。

‍

尝试类型：int f(arr, L, R)​：在(L,R)范围上我先手情况下最大分数返回

这种试法的base case 是if(L == R){ return arr[L];}​

题目已经给了只能拿最左或者最右的牌，那么就每个都试一下，我会选对我最有利的情况

​Math.max((arr[L] + s(arr, L+1, R)), (arr[R] + s(arr, L, R-1)));​

后手函数int s(arr, L, R)​：在(L,R)范围上后手情况下最大分数返回

base case 是if(L == R){ return 0;}​只有一个被别人拿走了

别人拿走的情况俩个都试下，对方会决定对我最不利的情况

​Math.min(f(arr, L+1, R), f(arr, L, R-1));​

6.给你一个栈，请你逆序这个栈，不能申请额外的数据结构，只能使用递归函数如何实现?

用一个函数f，这个函数实现只把栈底的数取出，其他的数继续在栈里

这个问题本质是使用了递归函数在系统栈里能保存信息的机制，进行栈的逆序

7.规定1和A对应、2和B对应、3和C对应….
那么一个数字字符串比如”111”，就可以转化为”AAA”、”KA”和”AK”。
给定一个只有数字字符组成的字符串str，返回有多少种转化结果。

这种使用从左往右尝试的方法，跟4.是一个思想，假设0 ~ i-1上的数已经确定转化结果了，从i位置往后开始尝试。
①如果i位置是为0，那么就是0种可能不可能转化成其他字符，也不能和下个数组成新的字符，所以直接返回0即可
②如果i位置上的数在3~9​之间，没办法和i+1位置上的数结合转化，因为超出26个字母了，所以直接转化
③如果i位置上的数在1~2​之间，1的话是总可以和i+1上的数结合转化的，所以是1的话有俩种决定，一是自己转化，二是和i+1的数一起转化
④而2要看i+1上的数是不是在0~6​，如果是可以一起转化，不是的话就只能自己转化，所以2还要讨论i+1上的值才能分为俩种决定

8.给定两个长度都为N的数组weights和values，weights[i]和values[i]分别代表i号物品的重量和价值。给定一个正数bag，表示一个载重bag的袋子，你装的物品不能超过这个重量。返回你能装下最多的价值是多少?

一样是从左往右试，0号要或不要，1号要或不要…i号要或不要

尝试的过程中要找到可变参数形式最简单（值可以表达不用链表），可变参数数量最简单

9.N皇后问题

N皇后问题是指在N*N的棋盘上要摆N个皇后，要求任何两个皇后不同行、不同列，也不在同一条斜线上。
给定一个整数n，返回n皇后的摆法有多少种。
n=1，返回1。
n=2或3，2皇后和3皇后问题无论怎么摆都不行，返回0
n=8，返回92

常数时间可以通过位运算优化
